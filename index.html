1. Présentation

Qu'est ce que WAAPI ?

WAAPI est l’abréviation de Web Animation API ou Web Animation Application Programming Interface. C’est une API native en Javascript qui est basée sur les animations css et qui permet de les gérer directement en Javascript. Grâce à cette API, il est possible de construire des animations et d’avoir un contrôle sur leur lecture. Elle combine la puissance des performances CSS et les avantages du Javascript.

Après Adobe Flash Player (1996), SMIL : Synchronized Multimedia Integration Language (1998) , JQuery (2006) et les animations CSS (2009), l'API native Javascript WAAPI apparait en 2013 et propose aux développeurs web des animations puissantes et performantes.

Le principal intérêt de WAAPI est de se passer d'une bibliothèque d'animation externe et de se servir directement des fonctions natives du navigateur.

2 Modèles

Il y a deux modèles à prendre en compte pour maîtriser cette API : le Timing Model (modèle de chronométrage) et l’Animation Model (modèle d’animation). Le Timing Model va gérer la durée de l’animation, le début, la fin et les autres étapes de l’animation. L’Animation Model gère quant à lui toutes les propriétés CSS d’une animation.

Comment utiliser l'API ?

Pour utiliser l’API, il faut utiliser la méthode JavaScript “animate”.

Elle prend 2 paramètres qui correspond chacun à un modèle :
- un tableau composé d'objet avec les différentes propriétés CSS que l'on veut utiliser pour animer.
- Le second paramètre est la timeline, définit par un objet qui peut prendre différentes options (le délai, l'itération, la durée, etc)

Avantages et Inconvénients

Avantages

  • Donne aux développeurs d'avantages de contrôle sur les animations CSS
  • Performances
  • Possibilité d'inverser, de mettre en pause, d'accélerer les animations.
  • Accès à toutes les animations
  • Natif

Concurrence

Il existe des alternatives pour construire des animations.
Par exemple la librairie Javascript JQuery propose des fonctionnalités d'animation. On y trouve les fonctions fadeIn(), fadeOut() qui permettent de faire des fondus d'apparitions d'elements ou encore la fonction animate() qui permet d'animer des propriétés CSS.

Pour faire des animations plus complexes, il est possible d'utiliser la librairie GreenSock. Pour une animation de SVG Morphing, cela demande beaucoup de connaissance, il est donc préferable de réaliser l'animation avec la librairie GreenSock, plutôt qu'en utilisant WAAPI.

3. Exemple

Ci-dessus une animation d'étoiles filantes, réalisée avec la Web Animation API

Comment faire cette animation ?

Présentation du code

Tout d'abord, il faut définir plusieurs variables tels que le nombre d'étoiles pour une animation, la durée maximale d'apparition d'une étoile ,la taille du "cadre" de l'animation. 




const starCount = 500;

const maxTime = 31;

const width = 600;

const height = 600;

Il faut aussi selectionner la balise qui va contenir l'animation. Ici ça sera la balise avec l'id "universe" 




const universe = document.getElementById('universe');

La balise HTML : 




<div id="universe"><div>

Pour chaque étoile, on lui attribue une position sur l'axe Y et une vitesse puiss on lui crée une balise div 




let ypos = Math.round(Math.random() * height);

let star = document.createElement('div');

let speed = 1000 * (Math.random() * maxTime + 1);

Ensuite on ajoute une classe "star" à l'étoile pour la stylisée, 




star.setAttribute('class', 'star');

On ajoute l'étoile (la balise div que l'on vient de créer) à l'univers (balise div avec l'id "universe")




universe.appendChild(star);

Fonction animate

Maintenant, on va voir la partie qui nous interresse le plus : la méthode JS "animate"

Comme énoncé précedemment la fonction animate a deux modèles : L'Animation Model et le Timing Model

Le Timing Model

Pour notre exemple on a : 




[
    {transform: 'translate3d(' + width + 'px, ' + ypos + 'px, 0)'}
    // Première keyframe,

    {transform: 'translate3d(-' + Math.random() * 256 + 'px, ' + ypos + 'px,
    0)'} // Deuxième keyframe
]

Ce tableau comporte les images clès (keyframes) qui sont défini ici par 2 objets, le premier objet contient les propriétés CSS pour la première keyframe (début de l'animation). On place l'étoile tout à droite à une hauteur défini aléatoirement par ypos.

La deuxième keyframe correspond à la position de l'étoile à la fin de l'animation

Timing Model



{
    delay: Math.random() * -speed,
    duration: speed,
    iterations: 1000
}

Dans cette objet, on définit le délai "delay" qui permet de démarrer l'animation en attendant un temps donné.
La durée de l'animation "duration", ici égale à la vitesse speed.
Et on choisit le nombre de fois que se répète l'animation avec "iterations". L'animation d'une étoile se répètera 1000 fois.
